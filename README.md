# 房坪系統

本專案是本機端的雙北實價登錄房市分析儀表板。可匯入 `.xlsx`、`.xls`、`.csv` 檔案，將成交資料轉成常用的「萬 / 坪」口徑，並提供地圖、行政區、房型、總價級距、坪數級距與車位分析。

## 本機執行

```powershell
npm install
npm run dev
```

預設會開啟 `http://127.0.0.1:3000`。

## 產出 HTML

```powershell
npm run build
```

成果會在 `dist/index.html`。因為 Vite 已設定 `base: './'`，可以保留整個 `dist` 資料夾後直接開啟 `index.html`。

此專案資料夾名稱含中文，Vite 在部分 Windows/Node 組合下會於 render 階段無聲退出；因此 `npm run build` 會先複製到暫存英文路徑建置，再把 `dist` 複製回本專案。

## 支援資料

目前優先支援政府實價登錄常見欄位：

- `鄉鎮市區`
- `土地位置建物門牌`
- `交易年月日`
- `總價元`
- `單價元平方公尺`
- `建物移轉總面積平方公尺`
- `車位移轉總面積平方公尺`
- `車位總價元`
- `建物現況格局-房`
- `建物現況格局-衛`
- `移轉層次`
- `建築完成年月`
- `編號`

也保留對 Gemini 原型中「建案、案名、社區、房型、格局、車位價格」等欄位名稱的相容。

## 注意事項

- 地圖定位會呼叫 ArcGIS geocoding 服務，地址會送出查詢；若資料含內部案源或敏感地址，建議先關閉地圖或改成本機座標資料。
- `dist` 已包含 Tailwind 產出的 `app.css` 與 Leaflet 樣式，開啟成果不需要再從 CDN 載入 CSS。
- Excel 解析套件 `xlsx` 目前有 npm audit 警示；若要產品化，建議改用後端解析或評估替代套件。
- 目前資料保存在瀏覽器狀態中，重新整理會消失；若要長期追蹤，下一步應加入資料庫與匯入批次紀錄。
